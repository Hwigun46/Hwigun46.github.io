# 클라우드활용 2주차: AWS Management Console과 클라우드 운영 환경

## 개념 정리

### AWS Management Console

AWS Management Console은 웹 브라우저에서 AWS 서비스를 관리할 수 있는 그래픽 기반 관리 도구다. 사용자는 명령어를 입력하지 않아도 화면에서 서비스를 검색하고, 리소스를 생성하고, 설정을 변경할 수 있다.

처음 AWS를 배우는 단계에서는 콘솔을 통해 서비스의 위치와 기본 구조를 익히는 것이 중요하다. 콘솔은 AWS 서비스를 눈으로 확인하면서 접근할 수 있기 때문에 클라우드 운영 환경에 익숙해지는 출발점이 된다.

### 클라우드 운영 환경

클라우드 운영 환경은 서버 한 대만 관리하는 것이 아니라 여러 서비스와 리소스를 함께 관리하는 구조다. EC2, S3, IAM, VPC 같은 서비스가 각각 독립적으로 존재하지만 실제 시스템에서는 서로 연결되어 사용된다.

예를 들어 EC2 인스턴스를 만들 때는 컴퓨팅 자원만 선택하는 것이 아니다. 리전, 네트워크, 보안 그룹, 키 페어, 스토리지 같은 설정도 함께 고려해야 한다.

### 서비스 검색

AWS 콘솔에서는 검색창을 통해 필요한 서비스를 찾을 수 있다. EC2, S3, IAM, VPC처럼 자주 사용하는 서비스는 검색해서 접근하는 방식이 가장 기본적이다.

서비스 이름을 외우는 것보다 중요한 것은 각 서비스가 어떤 영역에 속하는지 구분하는 것이다. EC2는 컴퓨팅, S3는 스토리지, IAM은 권한 관리, VPC는 네트워크 영역에 해당한다.

### 리전

리전은 AWS 데이터센터가 위치한 지리적 지역 단위다. 사용자는 서비스를 생성할 때 어느 리전에 리소스를 만들지 선택해야 한다.

리전 선택은 성능, 비용, 법적 요구사항, 장애 대응과 연결된다. 사용자가 한국에 많다면 서울 리전을 선택하는 것이 지연 시간을 줄이는 데 유리할 수 있다.

### 가용 영역

가용 영역은 하나의 리전 안에 존재하는 독립적인 데이터센터 묶음이다. 여러 가용 영역을 사용하면 하나의 데이터센터에 문제가 생겨도 다른 영역에서 서비스를 계속 운영할 수 있다.

처음에는 리전은 “지역”, 가용 영역은 “그 지역 안의 분리된 데이터센터 단위”로 이해하면 된다.

### 대시보드

대시보드는 서비스의 상태와 주요 정보를 한눈에 확인하는 화면이다. EC2 대시보드에서는 인스턴스, 보안 그룹, 키 페어, 볼륨 같은 정보를 볼 수 있고, S3 대시보드에서는 버킷과 객체를 확인할 수 있다.

대시보드는 단순한 메뉴 화면이 아니라 현재 클라우드에 어떤 자원이 만들어져 있는지 확인하는 출발점이다.

### AWS CLI

AWS CLI는 명령어를 통해 AWS 서비스를 관리하는 도구다. 콘솔이 화면을 통해 조작하는 방식이라면, CLI는 명령어로 리소스를 조회하거나 생성하거나 삭제하는 방식이다.

초기 학습에서는 콘솔이 더 직관적이지만, 반복 작업이나 자동화가 필요해지면 CLI가 중요해진다.

### AWS SDK

AWS SDK는 프로그래밍 언어에서 AWS 서비스를 사용할 수 있게 해주는 개발 도구 모음이다. Python, Java 같은 언어로 AWS 리소스를 제어할 때 사용한다.

콘솔은 사람이 직접 조작하는 방식이고, CLI는 명령어 기반 조작이며, SDK는 코드 기반 조작이다.

| 도구 | 사용 방식 | 특징 |
|---|---|---|
| Management Console | 웹 화면 | 처음 배우기 쉽고 시각적 |
| CLI | 명령어 | 반복 작업과 자동화에 유리 |
| SDK | 코드 | 애플리케이션에서 AWS 기능 사용 |

## 한 줄 정리

AWS Management Console은 AWS 서비스를 눈으로 확인하고 조작하면서 클라우드 운영 환경에 익숙해지는 가장 기본적인 출발점이다.
