# 클라우드활용 6주차: S3와 VPC 기초 이해

## 개념 정리

### S3

S3는 Simple Storage Service의 약자로, AWS에서 제공하는 객체 스토리지 서비스다. 파일을 서버의 디스크에 저장하는 방식이 아니라, 객체라는 단위로 저장한다.

이미지, 문서, 백업 파일, 로그 파일, 정적 웹 파일처럼 다양한 데이터를 저장할 수 있다. S3는 EC2처럼 서버를 직접 관리하는 서비스가 아니라, 저장 공간을 서비스 형태로 사용하는 방식이다.

### 객체 스토리지

객체 스토리지는 데이터를 파일 경로 중심으로 관리하는 전통적인 파일 시스템과 다르게, 객체 단위로 저장하고 관리하는 방식이다. 객체는 데이터 자체와 메타데이터, 고유한 키를 함께 가진다.

S3에서 파일 하나를 업로드하면 그것이 하나의 객체가 된다.

### 버킷

버킷은 S3에서 객체를 담는 가장 기본적인 저장 공간이다. 파일을 바로 S3 전체에 올리는 것이 아니라, 먼저 버킷을 만들고 그 안에 객체를 업로드한다.

버킷 이름은 고유해야 하며, 버킷은 특정 리전에 생성된다. 버킷 단위로 권한, 공개 여부, 암호화, 버전 관리 같은 설정을 적용할 수 있다.

### 객체

객체는 S3에 저장되는 실제 데이터 단위다. 이미지 파일, PDF 파일, 로그 파일 같은 데이터가 모두 객체가 될 수 있다.

객체는 키를 통해 구분된다. 키는 파일 이름처럼 보일 수 있지만, S3에서는 객체를 식별하는 이름 역할을 한다.

### 스토리지 클래스

스토리지 클래스는 데이터 접근 빈도와 비용에 따라 저장 방식을 선택하는 기능이다. 자주 접근하는 데이터는 빠르게 접근할 수 있는 클래스를 사용하고, 거의 접근하지 않는 백업 데이터는 더 저렴한 클래스를 사용할 수 있다.

| 스토리지 클래스 | 특징 | 적합한 데이터 |
|---|---|---|
| Standard | 자주 접근하는 데이터 | 일반 서비스 파일 |
| Standard-IA | 자주 접근하지 않지만 필요 시 빠른 접근 | 백업, 장기 보관 |
| Glacier | 장기 보관 중심 | 아카이브 데이터 |
| Intelligent-Tiering | 접근 패턴에 따라 자동 조정 | 접근 빈도를 예측하기 어려운 데이터 |

### 내구성

S3는 데이터를 여러 가용 영역에 중복 저장하여 높은 내구성을 제공한다. 내구성은 저장한 데이터가 손실되지 않을 가능성을 의미한다.

가용성이 “서비스를 사용할 수 있는가”에 가깝다면, 내구성은 “저장한 데이터가 사라지지 않는가”에 가깝다.

### 퍼블릭 접근

S3 버킷이나 객체는 설정에 따라 외부에서 접근 가능하게 만들 수 있다. 하지만 실수로 중요한 데이터를 공개하면 보안 사고로 이어질 수 있다.

S3를 사용할 때는 버킷이 공개되어야 하는지, 특정 사용자만 접근해야 하는지 먼저 구분해야 한다.

### 버킷 정책

버킷 정책은 S3 버킷에 적용하는 접근 제어 정책이다. 어떤 사용자가 어떤 객체에 접근할 수 있는지 JSON 형식으로 정의할 수 있다.

IAM 정책이 사용자나 역할 중심의 권한이라면, 버킷 정책은 버킷 자체에 적용되는 권한 설정으로 이해할 수 있다.

### VPC

VPC는 Virtual Private Cloud의 약자로, AWS 안에서 사용자가 직접 구성하는 가상 네트워크 공간이다. EC2 같은 리소스는 VPC 안에 배치된다.

VPC는 클라우드 안에 만드는 나만의 네트워크 영역으로 이해할 수 있다. IP 범위, 서브넷, 라우팅, 인터넷 연결 여부 등을 설정할 수 있다.

### 서브넷

서브넷은 VPC 안을 더 작은 네트워크 영역으로 나눈 것이다. 퍼블릭 서브넷과 프라이빗 서브넷으로 나누어 리소스의 외부 접근 여부를 구분할 수 있다.

웹 서버처럼 외부 사용자가 접근해야 하는 리소스는 퍼블릭 서브넷에 두고, 데이터베이스처럼 외부에서 직접 접근하면 안 되는 리소스는 프라이빗 서브넷에 두는 방식이 일반적이다.

### 퍼블릭 서브넷과 프라이빗 서브넷

퍼블릭 서브넷은 인터넷과 직접 연결될 수 있는 서브넷이다. 프라이빗 서브넷은 외부 인터넷에서 직접 접근하기 어렵게 구성한 서브넷이다.

이 구분은 클라우드 네트워크 보안의 기본이다. 모든 리소스를 외부에 공개하는 것이 아니라, 필요한 리소스만 공개하고 내부 리소스는 숨겨야 한다.

## 한 줄 정리

S3는 객체를 저장하는 서비스이고, VPC는 AWS 안에서 리소스가 배치되는 가상 네트워크 공간이다.
