# 클라우드활용 11주차: AWS 조직 및 계정 관리

이번 주차의 핵심은 AWS 계정을 하나만 사용하는 구조에서 벗어나, 여러 계정을 조직 단위로 관리하는 방식을 이해하는 것이다. 클라우드 환경이 커질수록 계정, 권한, 비용, 보안 정책을 분리해서 관리하는 능력이 중요해진다.

## AWS 계정

AWS 계정은 AWS 리소스를 사용할 수 있는 고유한 관리 단위이다.

하나의 AWS 계정 안에서 EC2, S3, RDS, IAM 같은 여러 서비스를 생성하고 사용할 수 있다. 계정 단위로 결제, 보안, 리소스 격리가 관리되기 때문에 AWS 운영에서 계정은 단순 로그인 단위보다 더 큰 의미를 가진다.

## 루트 사용자

루트 사용자는 AWS 계정을 만들 때 생성되는 최상위 권한 사용자이다.

루트 사용자는 모든 리소스와 설정에 접근할 수 있기 때문에 일반적인 운영 작업에는 사용하지 않는 것이 좋다. 일상적인 작업은 IAM 사용자나 역할을 만들어 필요한 권한만 부여하는 방식이 안전하다.

## 다계정 환경

다계정 환경은 하나의 AWS 계정에 모든 리소스를 몰아넣지 않고, 목적별로 여러 계정을 나누어 운영하는 방식이다.

예를 들어 개발 계정, 운영 계정, 보안 계정, 로그 계정, 테스트 계정을 분리하면 실수나 사고가 전체 환경으로 확산되는 것을 줄일 수 있다.

## 다계정 환경의 장점

다계정 구조를 사용하면 보안, 비용, 운영 관리가 더 명확해진다.

| 장점 | 의미 |
|---|---|
| 업무 분리 | 팀, 서비스, 프로젝트별로 계정을 나눌 수 있음 |
| 환경 분리 | 개발, 테스트, 운영 환경을 분리할 수 있음 |
| 보안 강화 | 민감 데이터와 일반 리소스를 분리할 수 있음 |
| 비용 관리 | 계정별 비용을 구분해 확인할 수 있음 |
| 영향 범위 제한 | 한 계정의 문제가 전체 조직으로 번지는 것을 줄일 수 있음 |
| 정책 관리 | 조직 단위로 공통 정책을 적용할 수 있음 |

## AWS Organizations

AWS Organizations는 여러 AWS 계정을 하나의 조직처럼 묶어 중앙에서 관리할 수 있게 해주는 서비스이다.

조직을 만들면 관리 계정 아래에 여러 멤버 계정을 연결할 수 있고, 계정들을 조직 단위로 나누어 관리할 수 있다. 이를 통해 결제 통합, 정책 적용, 계정 분류, 보안 통제를 중앙에서 수행할 수 있다.

## OU

OU는 Organizational Unit의 약자로, 조직 안에서 계정을 묶는 단위이다.

부서, 프로젝트, 서비스, 환경에 따라 OU를 나눌 수 있다. 예를 들어 `Production`, `Development`, `Security`, `Sandbox` 같은 OU를 만들고 각 계정을 배치하면 계정 구조를 더 체계적으로 관리할 수 있다.

## SCP

SCP는 Service Control Policy의 약자로, AWS Organizations에서 계정이나 OU가 사용할 수 있는 AWS 서비스와 작업 범위를 제한하는 정책이다.

SCP는 권한을 직접 부여하는 정책이 아니라, 허용 가능한 권한의 상한선을 정하는 정책이다. 어떤 사용자가 IAM 정책으로 권한을 가지고 있더라도 SCP에서 제한된 작업은 수행할 수 없다.

## SCP와 IAM 정책 차이

| 구분 | SCP | IAM 정책 |
|---|---|---|
| 적용 범위 | OU 또는 계정 전체 | 사용자, 그룹, 역할 |
| 역할 | 사용할 수 있는 권한의 최대 범위 제한 | 실제 세부 권한 부여 |
| 기본 관점 | 조직 차원의 제한 | 개별 사용자 권한 관리 |
| 핵심 | 어디까지 가능한지 제한 | 무엇을 할 수 있는지 허용 |

SCP는 조직 전체의 안전장치이고, IAM 정책은 실제 사용자나 역할의 세부 권한 설정이라고 볼 수 있다.

## 정책 유형

AWS Organizations에서는 SCP 외에도 여러 정책을 사용할 수 있다.

| 정책 | 의미 |
|---|---|
| SCP | 서비스와 작업 범위 제한 |
| RCP | 리소스 접근 제한 |
| 백업 정책 | 계정별 백업 계획 중앙 관리 |
| 태그 정책 | 리소스 태그 일관성 유지 |
| 관리 정책 | 특정 기능이나 보안 구성을 중앙 통제 |

## 계정 초대와 생성

AWS Organizations에서는 새 계정을 만들거나 기존 AWS 계정을 조직으로 초대할 수 있다.

조직에 포함된 계정은 중앙 관리 구조 안에서 정책과 결제 관리 대상이 된다. 계정 수가 많아질수록 수동 관리보다 조직 구조와 정책 기반 관리가 필요해진다.

## 한 줄 정리

AWS Organizations는 여러 AWS 계정을 하나의 조직 구조로 묶고, OU와 SCP를 통해 보안·비용·정책 관리를 중앙화하는 서비스이다.
