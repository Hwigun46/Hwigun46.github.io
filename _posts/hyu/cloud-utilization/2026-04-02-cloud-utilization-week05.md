# 클라우드활용 5주차: EC2 기반 컴퓨팅 서비스와 인스턴스 운영

## 1. 이번 주차에서 다룬 내용

이번 주차에서는 AWS의 대표적인 컴퓨팅 서비스인 EC2를 학습했다. EC2는 AWS에서 가상 서버를 생성하고 운영하는 서비스이며, 이후 웹 서버 구성, 네트워크 설정, 보안 그룹 실습의 중심이 된다.

이번 주차에서 다룬 내용은 다음과 같다.

- EC2 인스턴스 개념
- EC2 인스턴스 구성 요소
- AMI의 의미와 종류
- 인스턴스 타입과 인스턴스 패밀리
- EBS와 Instance Store 차이
- EC2 요금제와 프리 티어
- EC2 인스턴스 생명주기
- 보안 그룹과 인바운드/아웃바운드 규칙
- EC2 인스턴스 생성과 접속 흐름

## 2. Background

- 가상 머신:
  물리 서버 위에서 독립된 서버처럼 실행되는 가상화된 컴퓨터다.

- AMI:
  EC2 인스턴스를 생성할 때 사용하는 OS와 소프트웨어 환경의 이미지다.

- 인스턴스 타입:
  CPU, 메모리, 네트워크 성능 등 EC2 사양을 결정하는 조합이다.

- EBS:
  EC2에 연결되는 네트워크 기반 블록 스토리지다.

- 보안 그룹:
  EC2 인스턴스 수준에서 트래픽을 제어하는 가상 방화벽이다.

### 보충 설명

EC2를 “그냥 AWS 서버”라고만 이해하면 부족하다. EC2는 AMI, 인스턴스 타입, 키 페어, 스토리지, VPC, 서브넷, 보안 그룹이 결합되어 실행되는 가상 서버다. 즉, EC2 인스턴스 하나를 만들 때도 컴퓨팅, 스토리지, 네트워크, 보안 요소가 같이 결정된다.

## 3. 개념 정리

### 3.1 EC2 인스턴스

Amazon EC2는 AWS에서 제공하는 가상 머신 서비스다. EC2 인스턴스는 EC2 서비스에서 생성되는 가상 서버 한 대로 이해할 수 있다.

EC2의 주요 특징은 다음과 같다.

- 필요할 때 생성하고 필요 없을 때 삭제 가능
- Linux, Windows 등 다양한 OS 선택 가능
- 클릭 또는 CLI로 빠르게 배포 가능
- 온디맨드, 예약, 스팟 등 다양한 요금 옵션 제공

### 3.2 EC2 인스턴스 구성 요소

| 구성 요소 | 설명 | 이해 포인트 |
|---|---|---|
| AMI | OS와 소프트웨어 환경 | 인스턴스의 시작 이미지 |
| 인스턴스 타입 | CPU, 메모리, 네트워크 성능 조합 | 서버 사양 결정 |
| EBS 볼륨 | EC2에 붙는 디스크 | 데이터 저장 공간 |
| 보안 그룹 | 인스턴스 방화벽 | 포트와 IP 접근 제어 |
| 키 페어 | SSH/RDP 접속 인증 수단 | 접속 보안과 연결 |
| VPC/서브넷 | EC2가 위치하는 네트워크 | 인스턴스는 항상 VPC 안에 존재 |
| 사용자 데이터 | 최초 실행 시 실행되는 스크립트 | 초기 자동 설정에 사용 |

### 3.3 AMI

AMI는 EC2 인스턴스를 생성할 때 기반이 되는 이미지다. OS 설치 CD 같은 역할로 이해하면 된다.

| AMI 종류 | 설명 | 주의할 점 |
|---|---|---|
| AWS 공식 AMI | AWS가 관리하는 기본 OS 이미지 | 보안 패치와 업데이트가 반영됨 |
| Marketplace AMI | 서드파티 소프트웨어 포함 이미지 | 추가 라이선스 비용 가능 |
| Custom AMI | 사용자가 인스턴스를 커스터마이징 후 저장 | 같은 설정을 여러 인스턴스에 복제 가능 |

강의자료에는 사용자 정의 AMI를 만드는 예시로 다음 명령어가 제시된다.

```bash
aws ec2 create-image --instance-id i-0123456789abcdef0 --name "MyWebServerAMI"
```

이 명령어는 이미 설정된 EC2 인스턴스를 이미지로 저장해 이후 같은 환경의 인스턴스를 다시 만들 때 사용할 수 있다.

### 3.4 인스턴스 타입

인스턴스 타입은 EC2 인스턴스의 CPU, 메모리, 스토리지, 네트워크 성능을 결정한다.

예시는 다음과 같다.

```text
t3.micro
m5.large
c6g.xlarge
```

인스턴스 타입 이름은 대체로 `패밀리 + 세대 + 크기`의 형태로 이해할 수 있다.

| 인스턴스 패밀리 | 특징 | 사용 사례 |
|---|---|---|
| t 시리즈 | 저렴하고 탄력적인 범용형 | 소규모 웹서버, 개발/테스트 |
| m 시리즈 | 균형형 범용 | 일반 웹/애플리케이션 서버 |
| c 시리즈 | 컴퓨팅 최적화 | 고성능 웹 서버, 배치 처리 |
| r 시리즈 | 메모리 최적화 | DB 서버, 인메모리 캐시 |
| g/p/inf/trn | GPU/AI/ML 특화 | 학습, 추론, 그래픽 처리 |
| i/d/h | 스토리지 최적화 | 로그 수집, NoSQL, 분산 파일 시스템 |

### 3.5 EBS와 Instance Store

| 구분 | EBS | Instance Store |
|---|---|---|
| 저장 방식 | 네트워크 기반 블록 스토리지 | 물리 호스트에 붙은 임시 저장소 |
| 데이터 유지 | 인스턴스 중지 후에도 유지 가능 | 인스턴스 중지 시 데이터 소실 |
| 백업 | 스냅샷 가능 | 일반적으로 영구 저장 목적 아님 |
| 속도 | 안정적인 블록 스토리지 | 매우 빠른 I/O 가능 |
| 사용 목적 | 일반적인 디스크 | 임시 데이터, 캐시 등 |

EC2에서 데이터를 안전하게 유지하려면 EBS를 기준으로 생각해야 한다. Instance Store는 빠르지만 임시 저장소이기 때문에 중요한 데이터를 두면 위험하다.

### 3.6 EC2 생명주기와 보안 그룹

EC2 인스턴스는 다음 상태 흐름을 가진다.

```text
pending → running → stopping → stopped → terminated
```

| 상태 | 의미 | 비용/운영 포인트 |
|---|---|---|
| pending | 시작 준비 중 | 아직 완전히 실행되지 않음 |
| running | 실행 중 | 인스턴스 비용 발생 |
| stopping | 중지 중 | 상태 전환 중 |
| stopped | 중지됨 | EBS 비용은 남을 수 있음 |
| terminated | 종료됨 | 인스턴스 삭제, 데이터 손실 가능 |

보안 그룹은 EC2 인스턴스의 가상 방화벽이다. 기본적으로 모든 인바운드는 차단하고, 모든 아웃바운드는 허용한다. 허용 규칙만 작성할 수 있으며 상태 저장 방식으로 동작한다.

## 4. 내가 몰랐을 가능성이 높은 개념

| 개념 | 왜 중요한가 | 이해 포인트 |
|---|---|---|
| AMI | EC2 생성의 시작점 | OS와 초기 소프트웨어 환경 |
| 인스턴스 타입 | 성능과 비용 결정 | 워크로드에 맞는 타입 선택 필요 |
| EBS | 데이터 보존과 연결 | 중지해도 유지될 수 있음 |
| Instance Store | 임시 저장소 이해 | 중지 시 데이터 소실 |
| Security Group | EC2 접근 제어 핵심 | 인스턴스 단위 가상 방화벽 |
| Stateful | 보안 그룹 동작 이해 | 응답 트래픽은 자동 허용 |
| Terminate | 데이터 손실 가능 | 종료 전 EBS 삭제 여부 확인 필요 |

## 5. 실습하면서 내 것으로 만들 부분

이번 주차는 EC2 인스턴스를 실제로 생성하고 접속하는 흐름이 중요하다. 특히 AMI, 인스턴스 타입, 키 페어, 보안 그룹, 스토리지를 각각 왜 설정하는지 연결해야 한다.

```text
EC2 서비스 접속
→ AMI 선택
→ 인스턴스 타입 선택
→ 키 페어 생성/다운로드
→ 보안 그룹 설정
→ 스토리지 설정
→ 인스턴스 생성
→ 상태 확인
→ RDP 또는 SSH 접속
→ 실습 후 중지 또는 종료
```

### 5.1 파이프라인

1. EC2 콘솔에 접속한다.
2. AMI를 선택한다.
3. 인스턴스 타입을 선택한다.
4. 키 페어를 생성하고 안전한 위치에 저장한다.
5. 보안 그룹에서 필요한 포트만 허용한다.
6. EBS 스토리지 크기를 확인한다.
7. 인스턴스를 생성한다.
8. 인스턴스 상태가 `running`인지 확인한다.
9. Windows 인스턴스라면 RDP로 접속한다.
10. 실습 후 비용 방지를 위해 중지 또는 종료한다.

### 5.2 명령어 또는 코드 흐름

```bash
# 사용자 정의 AMI 생성 예시
aws ec2 create-image --instance-id i-0123456789abcdef0 --name "MyWebServerAMI"

# EC2 인스턴스 정보 조회
aws ec2 describe-instances

# 서울 리전의 EC2 인스턴스 조회
aws ec2 describe-instances --region ap-northeast-2
```

### 5.3 명령어별로 정리

| 명령어 | 언제 쓰는가 | 내가 이해한 의미 |
|---|---|---|
| `aws ec2 create-image --instance-id ... --name ...` | 기존 인스턴스를 AMI로 저장할 때 | 설정된 서버 환경을 이미지로 보존 |
| `aws ec2 describe-instances` | EC2 인스턴스 정보를 확인할 때 | 인스턴스 상태와 설정 조회 |
| `aws ec2 describe-instances --region ap-northeast-2` | 특정 리전의 인스턴스를 볼 때 | 서울 리전 기준으로 EC2 조회 |

### 5.4 헷갈렸던 지점

- EC2 인스턴스는 AMI만으로 만들어지는 것이 아니라 네트워크와 보안 그룹까지 함께 필요하다.
- 인스턴스를 중지해도 EBS 비용은 남을 수 있다.
- 종료는 중지보다 강한 동작이며 데이터 손실 위험이 있다.
- 보안 그룹은 Deny 규칙이 아니라 Allow 규칙 중심이다.
- Windows 접속은 RDP 포트 3389가 필요하지만, 전 세계에 열면 위험하다.

| 개념 A | 개념 B | 차이점 |
|---|---|---|
| Stop | Terminate | Stop은 중지, Terminate는 삭제 |
| EBS | Instance Store | EBS는 지속 저장, Instance Store는 임시 저장 |
| AMI | Instance Type | AMI는 OS 이미지, Instance Type은 서버 사양 |
| Inbound | Outbound | Inbound는 들어오는 트래픽, Outbound는 나가는 트래픽 |

### 5.5 나중에 다시 볼 포인트

- EC2 인스턴스를 만들 때 필요한 구성 요소를 설명할 수 있는가?
- AMI와 인스턴스 타입의 차이를 구분할 수 있는가?
- EBS와 Instance Store 중 중요한 데이터를 어디에 둬야 하는가?
- 보안 그룹에서 22, 80, 443, 3389 포트가 각각 어떤 의미인지 설명할 수 있는가?

## 6. 보충 설명

* 자료 기반 내용:
  * EC2는 AWS에서 제공하는 가상 머신 서비스다.
  * EC2 인스턴스는 AMI, 인스턴스 타입, EBS, 보안 그룹, 키 페어, 네트워크 환경과 연결된다.
  * EC2는 온디맨드, 예약/Savings Plans, 스팟 요금 모델을 사용할 수 있다.
  * 보안 그룹은 인스턴스 수준의 가상 방화벽이다.

* 보충 설명:
  * 실습용 EC2는 종료 전 EBS 삭제 여부를 확인해야 한다.
  * SSH나 RDP 포트는 가능하면 본인 IP로만 제한해야 한다.

* 확실하지 않음:
  * 5주차 자료만으로는 실제 실습 계정에서 사용한 정확한 AMI ID와 보안 그룹 이름은 확인되지 않는다.
