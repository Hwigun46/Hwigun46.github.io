# 클라우드활용 5주차: EC2 기반 컴퓨팅 서비스 이해

## 개념 정리

### EC2

EC2는 Elastic Compute Cloud의 약자로, AWS에서 가상 서버를 생성하고 실행할 수 있게 해주는 컴퓨팅 서비스다. 사용자는 물리 서버를 직접 구매하지 않고도 AWS 클라우드 안에서 서버를 만들 수 있다.

EC2 인스턴스는 클라우드에서 실행되는 가상 머신으로 이해할 수 있다. 웹 서버, 애플리케이션 서버, 테스트 서버처럼 다양한 용도로 사용할 수 있다.

### 인스턴스

인스턴스는 EC2에서 생성한 가상 서버 한 대를 의미한다. 인스턴스를 만들 때는 운영체제, 성능, 저장 공간, 네트워크, 보안 설정을 함께 선택해야 한다.

EC2는 단순히 서버 하나를 켜는 기능이 아니라, 여러 설정 요소가 결합된 컴퓨팅 자원이다.

### AMI

AMI는 Amazon Machine Image의 약자로, EC2 인스턴스를 시작할 때 사용하는 기본 이미지다. 운영체제와 기본 소프트웨어 구성이 포함된 서버 템플릿처럼 볼 수 있다.

예를 들어 Ubuntu AMI를 선택하면 Ubuntu 기반 서버가 생성되고, Windows AMI를 선택하면 Windows 기반 서버가 생성된다.

### 인스턴스 타입

인스턴스 타입은 EC2 인스턴스의 CPU, 메모리, 네트워크 성능, 스토리지 성능을 결정하는 조합이다. `t3.micro`, `m5.large`, `c6g.xlarge` 같은 형식으로 표현된다.

인스턴스 타입은 서버의 용도에 맞게 선택해야 한다. 단순 테스트 서버와 대규모 데이터 처리 서버가 같은 성능을 가질 필요는 없다.

| 인스턴스 계열 | 특징 | 적합한 사용 예 |
|---|---|---|
| 범용 | CPU와 메모리 균형 | 웹 서버, 일반 애플리케이션 |
| 컴퓨팅 최적화 | CPU 성능 중심 | 연산 작업, 고성능 처리 |
| 메모리 최적화 | 메모리 용량 중심 | 대용량 데이터 처리 |
| 스토리지 최적화 | 디스크 입출력 중심 | 대규모 로그, 데이터 처리 |

### EBS

EBS는 Elastic Block Store의 약자로, EC2 인스턴스에 연결해서 사용하는 블록 스토리지다. 쉽게 말하면 EC2에 붙이는 디스크 역할을 한다.

EBS는 인스턴스와 분리된 저장 공간으로 볼 수 있다. 인스턴스를 중지해도 EBS 볼륨의 데이터는 유지될 수 있으며, 스냅샷을 통해 백업할 수도 있다.

### 인스턴스 스토어

인스턴스 스토어는 EC2 인스턴스에 직접 붙어 있는 임시 저장 공간이다. 성능은 좋을 수 있지만 인스턴스가 종료되면 데이터가 사라질 수 있다.

따라서 중요한 데이터를 저장하기보다는 임시 파일이나 캐시처럼 사라져도 되는 데이터에 적합하다.

### 보안 그룹

보안 그룹은 EC2 인스턴스에 적용되는 가상 방화벽이다. 어떤 포트로 접근을 허용할지, 어떤 IP에서 접근할 수 있는지를 설정한다.

예를 들어 SSH 접속을 위해 22번 포트를 열거나, 웹 접속을 위해 80번 또는 443번 포트를 열 수 있다. 불필요한 포트를 열어두면 보안 위험이 커진다.

### 키 페어

키 페어는 EC2 인스턴스에 SSH로 접속할 때 사용하는 인증 수단이다. 공개 키는 AWS에 등록되고, 개인 키는 사용자가 보관한다.

개인 키를 잃어버리면 인스턴스 접속이 어려워질 수 있고, 개인 키가 노출되면 서버 접근 권한이 노출될 수 있다.

### 사용자 데이터

사용자 데이터는 EC2 인스턴스가 처음 시작될 때 자동으로 실행되는 스크립트다. 서버 시작과 동시에 패키지를 설치하거나 웹 서버를 실행하는 데 사용할 수 있다.

사용자 데이터는 서버 초기 설정을 자동화하는 기본적인 방법이다.

## 한 줄 정리

EC2는 AWS에서 가상 서버를 만드는 서비스이며, 인스턴스는 AMI, 인스턴스 타입, 스토리지, 네트워크, 보안 설정이 함께 구성되어 동작한다.
