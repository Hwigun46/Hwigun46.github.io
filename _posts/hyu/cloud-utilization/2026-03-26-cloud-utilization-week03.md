# 클라우드활용 3주차: AWS 비용 구조와 과금 모델

## 개념 정리

### 클라우드 과금 구조

AWS 비용은 사용한 리소스의 종류와 사용량에 따라 결정된다. 서버를 얼마나 오래 실행했는지, 데이터를 얼마나 저장했는지, 네트워크로 데이터를 얼마나 전송했는지에 따라 비용이 달라진다.

클라우드의 장점은 필요한 만큼 사용할 수 있다는 점이지만, 동시에 사용량을 제대로 관리하지 않으면 비용이 계속 발생할 수 있다는 뜻이기도 하다.

### 사용량 기반 과금

사용량 기반 과금은 클라우드 비용 구조의 기본이다. EC2 인스턴스를 실행하면 실행 시간에 따라 비용이 발생하고, S3에 데이터를 저장하면 저장 용량에 따라 비용이 발생한다.

이 구조에서는 리소스를 생성하는 것만큼 종료하거나 삭제하는 것도 중요하다. 실습용 리소스를 켜둔 채 방치하면 작은 비용이라도 계속 누적될 수 있다.

### 온디맨드 요금

온디맨드 요금은 약정 없이 사용한 만큼 비용을 지불하는 방식이다. 필요한 때에 서버를 실행하고 사용한 시간만큼 비용을 내는 구조다.

가장 유연하지만 장기간 계속 사용하는 경우에는 다른 요금 모델보다 비쌀 수 있다. 사용량을 예측하기 어려운 초기 서비스나 짧은 테스트 환경에 적합하다.

### 예약 인스턴스

예약 인스턴스는 일정 기간 사용을 약정하고 할인받는 방식이다. 1년 또는 3년처럼 장기간 사용할 리소스가 예측될 때 비용을 줄일 수 있다.

항상 켜져 있어야 하는 서버처럼 사용 패턴이 안정적인 경우에 적합하다. 대신 약정이 들어가기 때문에 사용량이 자주 바뀌는 환경에는 부담이 될 수 있다.

### Savings Plans

Savings Plans는 일정 사용량을 약정하고 할인을 받는 요금 모델이다. 예약 인스턴스와 비슷하게 장기 사용 비용을 줄이는 데 사용된다.

핵심은 앞으로 일정 수준 이상의 컴퓨팅 사용이 계속될 것이라고 판단될 때 비용을 최적화하는 방식이라는 점이다.

### 스팟 인스턴스

스팟 인스턴스는 AWS의 남는 컴퓨팅 자원을 저렴하게 사용하는 방식이다. 가격은 저렴하지만 AWS가 해당 자원을 회수할 수 있기 때문에 언제든 중단될 가능성이 있다.

따라서 웹 서버처럼 계속 안정적으로 실행되어야 하는 작업보다는 중단되어도 다시 실행할 수 있는 배치 작업, 테스트, 데이터 처리 작업에 더 적합하다.

### 프리 티어

프리 티어는 AWS를 처음 사용하는 사용자가 일부 서비스를 제한된 범위 안에서 무료로 사용할 수 있게 해주는 제도다. 학습 단계에서는 프리 티어를 활용해 기본 서비스를 경험할 수 있다.

다만 무료 사용량을 초과하거나 프리 티어 대상이 아닌 서비스를 사용하면 비용이 발생할 수 있다. 무료라는 표현만 믿고 리소스를 방치하면 안 된다.

### 비용 관리 도구

AWS에서는 비용을 확인하고 예산을 설정하는 도구를 제공한다. 비용 대시보드, 예산 설정, 알림 기능을 활용하면 예상보다 많은 비용이 발생하는 상황을 줄일 수 있다.

초기 학습 단계에서도 비용 확인 습관은 중요하다. 클라우드는 자원을 빠르게 만들 수 있는 만큼 비용도 빠르게 발생할 수 있다.

| 요금 모델 | 특징 | 적합한 상황 |
|---|---|---|
| 온디맨드 | 약정 없이 사용한 만큼 지불 | 테스트, 단기 사용, 예측 어려운 서비스 |
| 예약 인스턴스 | 장기 약정으로 할인 | 오래 켜두는 서버 |
| Savings Plans | 일정 사용량 약정으로 할인 | 지속적인 컴퓨팅 사용 |
| 스팟 인스턴스 | 저렴하지만 중단 가능 | 배치 작업, 실험, 재시작 가능한 작업 |
| 프리 티어 | 제한된 무료 사용 | 학습과 기초 실습 |

## 한 줄 정리

AWS 비용 구조의 핵심은 사용한 만큼 비용이 발생한다는 점이며, 클라우드를 제대로 쓰려면 리소스 생성보다 비용 관리 습관을 먼저 익혀야 한다.
