# 클라우드활용 1주차: AWS 클라우드 컴퓨팅 개요와 글로벌 인프라 구조

## 개념 정리

### 클라우드 컴퓨팅

클라우드 컴퓨팅은 서버, 스토리지, 네트워크, 데이터베이스 같은 IT 자원을 직접 구매해서 소유하는 대신 인터넷을 통해 필요한 만큼 사용하는 방식이다. 사용자는 물리 서버를 직접 설치하지 않아도 되고, 필요한 시점에 자원을 생성하거나 줄일 수 있다.

핵심은 “서버를 어디에 두는가”보다 “인프라를 어떻게 사용하는가”에 있다. 클라우드는 인프라를 제품처럼 구매하는 방식이 아니라 서비스처럼 사용하는 방식에 가깝다.

### 온디맨드

온디맨드는 필요한 순간에 바로 사용할 수 있다는 의미다. 클라우드에서는 서버나 저장 공간을 미리 준비해두지 않아도 필요할 때 생성할 수 있다.

이 개념은 클라우드의 기본 사용 방식이다. 사용자는 앞으로 얼마나 필요할지 완벽하게 예측하지 않아도 되고, 현재 필요한 만큼 먼저 사용한 뒤 상황에 따라 조정할 수 있다.

### 사용량 기반 과금

클라우드는 사용한 만큼 비용을 지불하는 구조를 가진다. 서버를 켜둔 시간, 저장한 데이터의 양, 전송한 데이터의 양처럼 실제 사용량이 비용과 연결된다.

이 방식은 초기 투자 비용을 줄일 수 있다는 장점이 있다. 하지만 리소스를 만들고 방치하면 사용하지 않는 것처럼 보여도 비용이 계속 발생할 수 있다. 클라우드는 자동으로 저렴한 환경이 아니라, 사용량을 관리할 수 있을 때 비용 효율적인 환경이다.

### 온프레미스

온프레미스는 기업이나 기관이 자체적으로 서버, 네트워크 장비, 저장 장치를 보유하고 운영하는 방식이다. 물리 장비를 직접 구매하고 설치하며, 장애 대응과 유지보수도 직접 담당한다.

온프레미스는 인프라를 직접 통제할 수 있다는 장점이 있지만, 초기 비용이 크고 확장 속도가 느리다. 서버가 부족하면 장비를 새로 구매해야 하고, 사용량이 줄어도 이미 구매한 장비는 그대로 남는다.

### 클라우드와 온프레미스 비교

| 구분 | 온프레미스 | 클라우드 |
|---|---|---|
| 인프라 소유 | 직접 소유 | 클라우드 제공자의 인프라 사용 |
| 초기 비용 | 큼 | 상대적으로 작음 |
| 확장 방식 | 장비 구매와 설치 필요 | 필요한 만큼 생성 가능 |
| 운영 부담 | 직접 관리 | 일부 관리 부담 감소 |
| 비용 구조 | 초기 투자 중심 | 사용량 기반 비용 중심 |

### 클라우드 도입 효과

클라우드를 사용하면 인프라를 빠르게 준비할 수 있고, 서비스 규모에 맞춰 자원을 늘리거나 줄일 수 있다. 또한 여러 지역에 서비스를 배포할 수 있어 글로벌 서비스 운영에도 유리하다.

다만 클라우드를 쓴다고 해서 모든 문제가 자동으로 해결되지는 않는다. 비용 관리, 권한 설정, 보안 설정, 리소스 삭제 같은 기본 운영 습관이 필요하다.

### AWS

AWS는 Amazon Web Services의 약자로, 아마존이 제공하는 클라우드 서비스 플랫폼이다. AWS는 컴퓨팅, 스토리지, 네트워킹, 데이터베이스, 보안, 모니터링 등 다양한 인프라 기능을 서비스 형태로 제공한다.

처음에는 모든 서비스를 외우기보다 영역별로 구분하는 것이 중요하다.

| 영역 | 대표 서비스 | 역할 |
|---|---|---|
| 컴퓨팅 | EC2, Lambda | 애플리케이션 실행 |
| 스토리지 | S3, EBS | 데이터 저장 |
| 네트워크 | VPC | 클라우드 네트워크 구성 |
| 보안 | IAM, WAF, GuardDuty | 권한 관리와 보안 보호 |
| 모니터링 | CloudWatch, CloudTrail | 상태 확인과 기록 추적 |

## 한 줄 정리

클라우드는 서버를 직접 소유하는 방식에서 벗어나, 필요한 IT 자원을 인터넷을 통해 필요한 만큼 사용하는 인프라 운영 방식이다.
