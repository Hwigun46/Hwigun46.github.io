# 클라우드활용 10주차: CloudWatch와 CloudTrail 모니터링

이번 주차의 핵심은 AWS 환경을 운영할 때 시스템 상태와 사용자 행위를 어떻게 확인하는지 이해하는 것이다. 클라우드 운영에서는 리소스를 만드는 것만큼 상태를 보고, 로그를 남기고, 이상 상황을 감지하는 것이 중요하다.

## CloudWatch

Amazon CloudWatch는 AWS 리소스와 애플리케이션을 모니터링하는 서비스이다.

CPU 사용률, 디스크 I/O, 네트워크 트래픽 같은 지표를 수집하고, 애플리케이션 로그나 시스템 로그를 모아 분석할 수 있다. 특정 조건이 발생하면 알람을 통해 이메일, SNS, Lambda 같은 다른 서비스와 연동할 수 있다.

CloudWatch는 클라우드 운영에서 시스템 상태를 보는 눈에 가깝다.

## 지표

지표는 시간 단위로 수집되는 수치 데이터이다.

예를 들어 EC2 인스턴스의 CPU 사용률, 메모리 사용량, 디스크 쓰기 횟수, 네트워크 트래픽 등이 지표가 될 수 있다. 지표는 시스템이 현재 정상적으로 동작하는지 판단하는 기준이 된다.

| 속성 | 의미 |
|---|---|
| Namespace | 지표가 속한 서비스 영역 |
| Dimension | 어떤 리소스의 지표인지 구분하는 값 |
| Statistic | 평균, 합계, 최대값 같은 계산 방식 |
| Period | 지표를 수집하는 주기 |

## 로그

로그는 시스템이나 애플리케이션에서 발생한 사건을 기록한 데이터이다.

지표가 숫자로 상태를 보여준다면, 로그는 어떤 일이 발생했는지를 문장이나 이벤트 단위로 보여준다. 예를 들어 애플리케이션 오류 메시지, 접근 기록, 시스템 이벤트 등이 로그에 해당한다.

## 알람

알람은 특정 지표가 정해진 조건을 만족했을 때 알림을 보내거나 자동 대응을 실행하는 기능이다.

예를 들어 CPU 사용률이 80%를 넘으면 관리자에게 알림을 보내거나, 특정 오류 로그가 반복되면 대응 작업을 실행하도록 구성할 수 있다. 알람은 문제가 발생한 뒤에 확인하는 방식이 아니라, 문제가 커지기 전에 감지하는 방식에 가깝다.

## CloudTrail

AWS CloudTrail은 AWS 계정 안에서 누가, 언제, 어떤 작업을 했는지 기록하는 서비스이다.

CloudWatch가 리소스 상태를 보는 도구라면, CloudTrail은 사용자 활동과 API 호출 이력을 추적하는 도구이다. EC2 인스턴스를 생성했는지, IAM 권한을 변경했는지, S3 버킷 정책을 수정했는지 같은 행위를 확인할 수 있다.

## CloudTrail 이벤트

CloudTrail 이벤트는 AWS에서 발생한 API 호출 기록이다.

| 이벤트 유형 | 의미 | 예시 |
|---|---|---|
| 관리 이벤트 | 리소스 생성, 삭제, 권한 변경 같은 관리 작업 | EC2 생성, IAM 사용자 생성 |
| 데이터 이벤트 | 리소스 내부 데이터에 접근하는 작업 | S3 객체 조회, Lambda 호출 |
| 인사이트 이벤트 | 비정상적인 API 호출 패턴 탐지 | 반복 실패 요청, 비정상 호출 증가 |

CloudTrail 로그에는 이벤트 시간, API 이름, 호출 주체, 접근 IP, 리전, 대상 리소스 같은 정보가 포함된다.

## CloudWatch와 CloudTrail 차이

| 구분 | CloudWatch | CloudTrail |
|---|---|---|
| 목적 | 시스템 상태 모니터링 | 사용자 행위 추적 |
| 데이터 | 지표, 로그, 알람 | API 호출 기록 |
| 질문 | 지금 시스템에 무슨 일이 일어나는가 | 누가 언제 무엇을 했는가 |
| 활용 | 운영, 장애 감지, 자동 대응 | 보안 감사, 변경 이력 추적 |

두 서비스는 역할이 다르지만 함께 사용해야 운영과 보안을 모두 확인할 수 있다.

## SNS

Amazon SNS는 알림을 여러 대상으로 전달하는 게시/구독 방식의 메시징 서비스이다.

CloudWatch 알람과 SNS를 연결하면 특정 조건이 발생했을 때 이메일이나 다른 대상으로 알림을 보낼 수 있다. 운영자가 직접 계속 대시보드를 보지 않아도 이상 징후를 알 수 있게 해주는 구조이다.

## SQS

Amazon SQS는 메시지를 큐에 저장해두고 소비자가 가져가 처리하는 메시징 서비스이다.

SNS가 여러 구독자에게 메시지를 밀어주는 Push 방식이라면, SQS는 메시지를 큐에 보관하고 소비자가 가져가는 Polling 방식에 가깝다.

| 구분 | SNS | SQS |
|---|---|---|
| 방식 | Push | Polling |
| 구조 | 게시/구독 | 생산자/소비자 |
| 메시지 지속성 | 낮음 | 높음 |
| 사용 흐름 | 알림 전송 | 작업 대기열 처리 |

## 한 줄 정리

CloudWatch는 시스템 상태를 보고, CloudTrail은 사용자 행위를 추적하며, 두 서비스를 함께 써야 AWS 운영과 보안을 제대로 관리할 수 있다.
