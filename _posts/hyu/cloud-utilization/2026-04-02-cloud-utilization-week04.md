# 클라우드활용 4주차: IAM을 활용한 사용자 권한 관리와 보안 기초

## 개념 정리

### IAM

IAM은 Identity and Access Management의 약자로, AWS에서 사용자와 권한을 관리하는 서비스다. 누가 어떤 AWS 리소스에 접근할 수 있는지, 어떤 작업을 할 수 있는지를 제어한다.

클라우드에서는 서버나 스토리지를 만드는 것만큼 권한 관리가 중요하다. 권한 설정이 잘못되면 사용자가 접근하면 안 되는 리소스에 접근하거나, 외부에 데이터가 노출될 수 있다.

### 접근 제어

접근 제어는 사용자가 어떤 대상에 접근할 수 있는지 제한하는 개념이다. AWS에서는 IAM을 통해 사용자, 그룹, 정책, 역할을 조합하여 접근 권한을 관리한다.

기본 방향은 필요한 사람에게 필요한 권한만 주는 것이다. 모든 사용자에게 관리자 권한을 주면 편해 보이지만, 보안 사고가 발생했을 때 피해 범위가 커진다.

### 루트 사용자

루트 사용자는 AWS 계정의 최상위 사용자다. 모든 권한을 가지고 있기 때문에 일상적인 작업에 사용하면 위험하다.

일반적인 운영에서는 루트 사용자를 최소한으로 사용하고, 필요한 작업은 IAM 사용자를 만들어 권한을 나누는 방식이 안전하다.

### IAM 사용자

IAM 사용자는 AWS 계정 안에서 생성하는 개별 사용자다. 사람마다 IAM 사용자를 따로 만들면 누가 어떤 작업을 했는지 구분할 수 있고, 사용자별로 필요한 권한만 줄 수 있다.

사용자 계정을 공유하면 책임 추적이 어려워진다. 클라우드 환경에서는 개인별 계정과 권한 분리가 기본이다.

### IAM 그룹

IAM 그룹은 여러 IAM 사용자를 묶어서 권한을 관리하는 단위다. 사용자마다 정책을 하나씩 붙이는 대신, 그룹에 정책을 연결하고 사용자를 그룹에 넣으면 관리가 쉬워진다.

예를 들어 개발자 그룹, 운영자 그룹, 읽기 전용 그룹처럼 역할에 따라 그룹을 나눌 수 있다.

### IAM 정책

IAM 정책은 어떤 작업을 허용하거나 거부할지 정의하는 문서다. 정책은 JSON 구조로 작성되며, Effect, Action, Resource 같은 요소를 사용한다.

정책의 핵심은 “누가 무엇을 어떤 대상에 할 수 있는가”를 명확히 지정하는 것이다.

| 요소 | 의미 |
|---|---|
| Effect | 허용할지 거부할지 결정 |
| Action | 허용하거나 거부할 작업 |
| Resource | 정책이 적용될 대상 리소스 |
| Condition | 조건부 제한 설정 |

### Allow와 Deny

Allow는 특정 작업을 허용하는 설정이고, Deny는 특정 작업을 명시적으로 거부하는 설정이다. AWS 권한 판단에서는 명시적인 거부가 강하게 적용된다.

따라서 정책을 볼 때 단순히 허용된 작업만 보는 것이 아니라, 거부 조건이 있는지도 함께 확인해야 한다.

### 최소 권한 원칙

최소 권한 원칙은 사용자에게 필요한 권한만 부여하는 보안 원칙이다. 예를 들어 S3 객체를 읽기만 하면 되는 사용자에게 S3 삭제 권한까지 줄 필요는 없다.

클라우드 보안의 기본은 넓은 권한을 편하게 주는 것이 아니라, 필요한 권한을 좁게 주는 것이다.

### IAM 역할

IAM 역할은 특정 사용자에게 고정된 권한을 부여하는 방식이 아니라, 필요한 상황에서 임시로 권한을 맡는 개념이다. EC2 인스턴스가 S3에 접근해야 할 때 직접 액세스 키를 넣는 대신 역할을 연결하는 방식이 사용될 수 있다.

초기 학습 단계에서는 사용자는 “사람에게 주는 권한”, 역할은 “서비스나 상황에 따라 맡는 권한”으로 이해하면 된다.

## 한 줄 정리

IAM은 AWS에서 사용자와 권한을 관리하는 핵심 서비스이며, 클라우드 보안의 출발점은 필요한 대상에게 필요한 권한만 주는 것이다.
