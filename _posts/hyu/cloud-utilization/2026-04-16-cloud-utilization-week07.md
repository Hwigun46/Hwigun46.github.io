# 클라우드활용 7주차: AWS 보안 기본 개념과 책임 공유 모델

## 개념 정리

### 클라우드 보안

클라우드 보안은 클라우드 환경에서 계정, 권한, 네트워크, 데이터, 애플리케이션을 안전하게 보호하는 개념이다. 클라우드는 인프라를 빠르게 만들 수 있는 만큼 잘못된 설정도 빠르게 위험으로 이어질 수 있다.

보안은 서비스를 배포한 뒤 마지막에 확인하는 항목이 아니라, 리소스를 만들 때부터 함께 고려해야 하는 기본 조건이다.

### 책임 공유 모델

책임 공유 모델은 AWS와 사용자가 보안 책임을 나누어 가진다는 개념이다. AWS는 클라우드를 운영하는 물리 인프라를 보호하고, 사용자는 클라우드 안에서 만든 리소스와 설정을 보호한다.

쉽게 말해 AWS는 데이터센터, 물리 서버, 기반 인프라를 책임지고, 사용자는 IAM 권한, S3 공개 설정, EC2 보안 그룹, 데이터 암호화 설정 같은 부분을 책임진다.

| 항목 | 주 책임 |
|---|---|
| 물리적 데이터센터 보안 | AWS |
| 물리 서버와 하드웨어 관리 | AWS |
| S3 버킷 접근 권한 설정 | 사용자 |
| EC2 운영체제 보안 패치 | 사용자 |
| 보안 그룹 포트 설정 | 사용자 |
| 데이터 암호화 설정 | 사용자 |

### 고객 책임

고객 책임은 사용자가 직접 설정하고 관리해야 하는 보안 영역이다. 대표적으로 IAM 권한, 네트워크 접근 제어, 데이터 암호화, 운영체제 패치, 애플리케이션 보안이 있다.

클라우드에서 보안 사고가 발생하는 경우 상당수는 클라우드 제공자의 물리 인프라 문제가 아니라 사용자의 설정 실수와 관련된다. 예를 들어 S3 버킷을 공개하거나, EC2의 SSH 포트를 넓게 열어두는 상황이 여기에 해당한다.

### IAM 보안

IAM은 사용자와 권한을 관리하는 보안 서비스다. 누가 AWS에 접근할 수 있는지, 어떤 서비스를 사용할 수 있는지 제어한다.

IAM 보안의 기본은 루트 사용자 사용을 줄이고, 사용자별 계정을 만들고, 최소 권한 원칙을 적용하는 것이다. 필요 이상의 권한은 관리 편의성은 높일 수 있지만 보안 위험을 크게 만든다.

### VPC 보안

VPC 보안은 클라우드 네트워크를 안전하게 분리하고 구성하는 것이다. 외부에 공개되어야 하는 리소스와 내부에서만 접근해야 하는 리소스를 구분해야 한다.

퍼블릭 서브넷과 프라이빗 서브넷을 나누는 이유도 여기에 있다. 모든 리소스를 인터넷에 직접 노출하는 구조는 단순하지만 위험하다.

### 보안 그룹

보안 그룹은 EC2 인스턴스에 적용되는 가상 방화벽이다. 어떤 포트와 어떤 IP에서 접근을 허용할지 설정한다.

예를 들어 SSH 접속을 위해 22번 포트를 열 수 있지만, 모든 IP에서 접근 가능하게 열어두면 위험하다. 필요한 포트만 필요한 대상에게 허용하는 것이 기본이다.

### WAF

WAF는 Web Application Firewall의 약자로, 웹 애플리케이션을 대상으로 하는 공격을 탐지하고 차단하는 보안 서비스다. SQL Injection, Cross-Site Scripting 같은 웹 공격을 막는 데 사용될 수 있다.

WAF는 네트워크 방화벽과 다르게 웹 요청의 내용을 보고 공격 패턴을 판단하는 데 초점이 있다.

### GuardDuty

GuardDuty는 AWS 계정과 워크로드에서 의심스러운 활동을 탐지하는 보안 서비스다. 비정상적인 API 호출, 의심스러운 네트워크 활동, 계정 침해 가능성 등을 탐지하는 데 사용된다.

초기 학습 단계에서는 GuardDuty를 “AWS 환경에서 이상 징후를 탐지하는 서비스”로 이해하면 된다.

### CloudTrail

CloudTrail은 AWS 계정에서 발생한 API 호출과 사용자 활동을 기록하는 서비스다. 누가 어떤 작업을 했는지 추적할 수 있기 때문에 보안 감사와 사고 분석에 중요하다.

보안에서는 문제가 발생한 뒤 원인을 추적할 수 있어야 한다. CloudTrail은 그 추적을 위한 기록을 남기는 역할을 한다.

### CloudWatch

CloudWatch는 AWS 리소스의 지표와 로그를 모니터링하는 서비스다. 시스템 상태, 성능, 로그를 확인하고 알람을 설정할 수 있다.

보안 관점에서도 이상한 트래픽 증가, 비정상적인 리소스 사용량, 장애 징후를 확인하는 데 도움이 된다.

## 한 줄 정리

AWS 보안의 핵심은 AWS가 모든 것을 대신 지켜주는 것이 아니라, AWS와 사용자가 각자의 책임 범위를 나누어 관리한다는 점이다.
