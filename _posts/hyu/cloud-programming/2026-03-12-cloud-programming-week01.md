# 클라우드프로그래밍 1주차: 클라우드 컴퓨팅의 기본 개념과 AWS 시작

## 1. 이번 주차에서 다룬 내용

클라우드프로그래밍의 첫 주차는 “클라우드가 무엇인가”를 잡는 시간이었다. 처음에는 클라우드를 단순히 인터넷 어딘가에 있는 서버 정도로 생각하기 쉽지만, 강의자료의 핵심은 그보다 명확하다.

클라우드는 필요한 IT 자원을 직접 구매하고 운영하는 대신, 인터넷을 통해 필요한 만큼 빌려 쓰고 사용량에 따라 비용을 지불하는 방식이다.

이번 주차에서 다룬 내용은 다음과 같다.

- 온프레미스와 클라우드 컴퓨팅의 차이
- 클라우드 컴퓨팅의 장점: 민첩성, 탄력성, 비용 절감
- IaaS, PaaS, SaaS의 차이
- 퍼블릭, 프라이빗, 하이브리드 클라우드
- AWS의 주요 서비스 분류
- AWS 계정 생성과 프리 티어 주의사항
- 리눅스 가상머신 실습 환경 준비

## 2. Background

- 서버: 요청을 받아 처리하고 응답을 제공하는 컴퓨터 또는 프로그램
- 데이터 센터: 서버와 네트워크 장비가 모여 있는 물리적 공간
- 네트워크: IT 자원들이 서로 통신할 수 있게 하는 연결 구조
- 가상화: 하나의 물리 자원을 여러 논리 자원처럼 나누어 쓰는 기술
- 운영체제: 하드웨어와 프로그램 사이에서 자원을 관리하는 소프트웨어
- 사용량 기반 과금: 시간, 용량, 요청 수 등에 따라 비용을 계산하는 방식

### 보충 설명

클라우드를 처음 배울 때 가장 흔한 오해는 “클라우드에는 서버가 없다”는 생각이다. 서버는 여전히 존재한다. 다만 사용자가 물리 서버의 위치, 전원, 냉각, 장애 교체를 직접 관리하지 않을 뿐이다.

클라우드는 서버가 사라진 구조가 아니라, 서버 관리의 많은 부분을 클라우드 공급자에게 넘긴 구조에 가깝다.

## 3. 개념 정리

### 3.1 온프레미스

온프레미스는 사용자가 자체 공간과 자체 자원을 이용해서 IT 인프라를 직접 구축하고 운영하는 방식이다.

직접 서버를 구매하고, 운영체제를 설치하고, 네트워크를 구성하고, 장애가 나면 직접 대응해야 한다. 제어권은 크지만 초기 비용과 운영 부담도 크다.

내가 이해한 방식으로 표현하면, 온프레미스는 집에서 모든 요리를 직접 하는 방식이다. 재료를 사고, 조리하고, 설거지하고, 주방 관리까지 직접 해야 한다. 자유도는 높지만 손이 많이 간다.

### 3.2 클라우드 컴퓨팅

클라우드 컴퓨팅은 인터넷을 통해 IT 자원을 온디맨드 방식으로 제공받고, 사용한 만큼 비용을 지불하는 서비스 방식이다.

여기서 중요한 표현은 두 가지다.

- 온디맨드: 필요할 때 바로 사용할 수 있다.
- Pay Per Use: 사용한 만큼 비용을 낸다.

서비스 트래픽은 항상 일정하지 않다. 처음부터 큰 서버를 사면 낭비가 생기고, 작은 서버만 준비하면 트래픽이 몰릴 때 장애가 난다. 클라우드는 필요한 만큼 자원을 늘리고 줄일 수 있게 해서 이 문제를 줄인다.

### 3.3 IaaS, PaaS, SaaS

클라우드 서비스 유형은 사용자가 어디까지 관리하는지에 따라 나뉜다.

| 구분 | 사용자가 주로 관리하는 것 | 예시로 이해하기 |
|---|---|---|
| IaaS | 서버, OS, 애플리케이션 설정 | 재료와 조리도구 일부를 빌려 직접 요리 |
| PaaS | 애플리케이션 코드 중심 | 조리 공간과 기본 도구가 준비된 상태에서 요리 |
| SaaS | 서비스 사용 | 완성된 음식을 주문해서 먹음 |

IaaS는 자유도가 높지만 관리할 것도 많다. SaaS는 관리 부담이 적지만 내부 구조를 직접 제어하기 어렵다.

### 3.4 클라우드 구축 모델

클라우드는 사용 범위에 따라 퍼블릭, 프라이빗, 하이브리드로 나눌 수 있다.

- 퍼블릭 클라우드: AWS, Azure, GCP처럼 외부 공급자의 자원을 사용하는 방식
- 프라이빗 클라우드: 특정 조직 내부에서 전용으로 사용하는 클라우드 방식
- 하이브리드 클라우드: 퍼블릭과 프라이빗을 함께 사용하는 방식

처음 공부할 때는 퍼블릭 클라우드만 생각하기 쉽지만, 실제 기업 환경에서는 규제, 보안, 기존 시스템 때문에 하이브리드 구조도 많이 고려된다.

### 3.5 AWS

AWS는 대표적인 퍼블릭 클라우드 서비스다. 강의자료에서는 컴퓨팅, 스토리지, 데이터베이스, 네트워크, 보안, 관리 도구 등 다양한 서비스가 등장한다.

처음부터 모든 서비스를 외울 필요는 없다. 중요한 것은 서비스 이름을 기능별로 묶어서 보는 것이다.

- EC2: 가상 서버
- S3: 객체 스토리지
- RDS: 관리형 관계형 데이터베이스
- VPC: 클라우드 네트워크
- IAM: 사용자와 권한 관리
- CloudWatch: 모니터링

## 4. 내가 몰랐을 가능성이 높은 개념

| 개념 | 왜 중요한가 | 이해 포인트 |
|---|---|---|
| 온디맨드 | 클라우드 사용 방식의 핵심 | 필요할 때 바로 자원을 할당받는 구조 |
| Pay Per Use | 비용 구조 이해에 필수 | 사용 시간, 용량, 요청 수에 따라 비용 발생 |
| 리전 | AWS 자원의 물리적 위치 이해 | 데이터센터가 모여 있는 지리적 단위 |
| 가용 영역 | 장애 대응 설계의 기본 | 하나의 리전 안에 있는 독립적인 데이터센터 단위 |
| 책임 공유 모델 | 클라우드 보안의 출발점 | AWS와 사용자가 나눠서 책임짐 |
| 서버리스 | Lambda 이해에 필요 | 서버가 없는 것이 아니라 서버 관리를 사용자가 덜 하는 것 |

## 5. 실습하면서 내 것으로 만들 부분

1주차 실습은 특정 서비스를 깊게 다룬다기보다, 앞으로 AWS와 리눅스 환경을 다루기 위한 준비에 가깝다. 그래서 실습 목표를 따로 세우기보다 “앞으로 계속 쓰게 될 기본 조작에 익숙해지는 것”이 중요하다.

### 5.1 파이프라인

1. AWS 계정을 만든다.
2. 프리 티어와 결제 정보를 확인한다.
3. AWS Management Console에 들어가 서비스 구성을 살펴본다.
4. 로컬에서 가상머신 기반 리눅스 환경을 준비한다.
5. Ubuntu를 설치하고 터미널을 사용해본다.
6. 기본 명령어를 입력하면서 리눅스 서버 환경에 익숙해진다.

여기서 핵심은 AWS 콘솔을 한 번 눌러봤다는 사실이 아니다. 앞으로 EC2에 SSH로 접속하면 대부분 터미널에서 작업하게 되므로, 리눅스 명령어를 읽고 쓰는 감각을 먼저 만들어야 한다.

### 5.2 기본 명령어 흐름

처음 리눅스 터미널을 열었을 때는 아래 흐름으로 익히는 것이 자연스럽다.

```bash
# 현재 날짜와 시간 확인
date

# 화면 정리
clear

# 명령어 사용법 확인
man date
man passwd

# 사용자 비밀번호 변경
passwd
```

이 명령어들은 화려하지 않지만, 서버를 다룰 때 가장 먼저 필요한 감각을 만든다. 특히 `man`은 모르는 명령어가 나왔을 때 문서를 직접 확인하는 습관과 연결된다.

### 5.3 명령어별로 정리

| 명령어 | 언제 쓰는가 | 내가 이해한 의미 |
|---|---|---|
| `date` | 서버의 현재 시간을 확인할 때 | 서버 시간이 정상인지 보는 가장 기본적인 명령어 |
| `clear` | 터미널 화면이 복잡해졌을 때 | 작업 공간을 정리하는 명령어 |
| `man 명령어` | 명령어 옵션과 사용법을 확인할 때 | 리눅스 내장 설명서를 여는 명령어 |
| `passwd` | 사용자 비밀번호를 바꿀 때 | 계정 인증 정보를 변경하는 명령어 |

### 5.4 같이 익혀두면 좋은 리눅스 기본 명령어

아래 명령어들은 강의 흐름을 이해하기 위한 보충 설명이다. 자료에 직접 나온 전체 명령어 목록이라고 단정하면 안 된다.

| 명령어 | 언제 쓰는가 | 내가 이해한 의미 |
|---|---|---|
| `pwd` | 현재 위치를 확인할 때 | 내가 지금 어느 디렉터리에 있는지 확인 |
| `ls` | 파일 목록을 볼 때 | 현재 디렉터리 안의 파일과 폴더 확인 |
| `cd 디렉터리` | 위치를 이동할 때 | 작업할 폴더로 이동 |
| `mkdir 이름` | 새 폴더를 만들 때 | 작업 공간 생성 |
| `cat 파일명` | 파일 내용을 빠르게 볼 때 | 짧은 설정 파일 확인에 유용 |

### 5.5 헷갈렸던 지점

처음에는 AWS 콘솔에서 버튼을 누르는 작업과 리눅스 명령어를 쓰는 작업이 따로 노는 것처럼 보일 수 있다. 하지만 EC2를 만들면 결국 리눅스 서버에 접속해서 설정을 해야 한다. 콘솔은 서버를 만드는 입구이고, 터미널은 서버를 실제로 다루는 작업 공간이다.

### 5.6 나중에 다시 볼 포인트

- AWS 프리 티어는 모든 서비스가 무료라는 뜻이 아니다.
- 클라우드 실습 후에는 자원 삭제와 비용 확인이 필요하다.
- 리눅스 기본 명령어는 뒤의 EC2, Docker, Kubernetes 실습에서 계속 사용된다.
- `man`으로 문서를 확인하는 습관을 들여야 한다.

## 6. 보충 설명

- 자료 기반 내용: 온프레미스, 클라우드 컴퓨팅, IaaS/PaaS/SaaS, 클라우드 구축 모델, AWS 서비스 라인업, AWS 가입, 리눅스 가상머신 설치
- 보충 설명: 클라우드는 서버가 없는 환경이 아니라, 서버 운영 부담을 줄이고 자원을 서비스처럼 사용하는 환경이다.
- 확실하지 않음: 실제 수업에서 AWS 계정을 직접 생성했는지, 시연 중심으로 진행했는지는 업로드된 강의자료만으로는 확실하지 않음.


## 7. 한 줄 요약

클라우드는 서버를 직접 소유하는 방식이 아니라, 필요한 IT 자원을 필요한 만큼 사용하고 운영 부담을 줄이는 컴퓨팅 방식이다.
